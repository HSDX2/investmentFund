# 基金投资邮件推送方案（分阶段 + 业界实践）

> 适用：投资基金自动化项目 · 个人 semi_auto · QQ 邮件推送  
> 最后更新：2026-05-21  
> 目标：**少而准、能行动、不制造焦虑**

---

## 一、设计原则（市面成熟做法）

参考蚂蚁/天天基金「提醒」、Morningstar 组合检视、海外 Robo（Betterment / Wealthfront）的通用共识：

| 原则 | 说明 |
|------|------|
| **行动优先** | 每封邮件最多 1 个主行动（买 / 卖 / 持有 / 分批第 N 笔） |
| **分层信息** | 3 秒能看懂结论 → 30 秒看表格 → 需要时再点开完整报告 |
| **事件驱动 + 固定节奏** | 日报/周报固定发；止损、限购、分批到期 **额外** 发，不重复堆叠 |
| **历史不代表未来** | 业绩数据必须带净值日期、口径；禁止「必涨」表述 |
| **频率上限** | 个人投资者：**每天 ≤1 封常规 + 偶尔 1 封紧急**；推荐类 **每周 ≤1 封** |
| **半自动边界** | 邮件 = 决策辅助 + 操作清单；**不下单、不代扣** |

## 四、A. 每日持仓简报（最重要，天天收）

**频率**：每个交易日 1 封（你现在 GitHub Actions 已做）  
**主题示例**：`【持仓】05-21 浮盈+12.9% · 建议：持有`

### 4.1 邮件结构（业界「倒金字塔」）

**第一层 — 3 秒结论（必放邮件最上）**

```
今日结论：持有
账户：621 元  |  今日：+0.42%  |  累计：+12.93%
需操作：无（270042 日定投 10 元照常）
```

**第二层 — 仅当有动作时展开「操作卡片」**

| 字段 | 是否推送 | 说明 |
|------|----------|------|
| 操作（加/减/持有） | ✅ | 1 行结论 |
| 基金代码 + 渠道 | ✅ | 去哪个 APP |
| 建议金额/份额 | ✅ 有操作时 | 无操作则省略整表 |
| 操作步骤 1～4 | ✅ 有操作时 | 复制现有 checklist |
| 置信度 / 理由 | ✅ 1 句 | 不要长文 |

**第三层 — 数据支撑（折叠/缩短）**

| 模块 | 建议 | 原因 |
|------|------|------|
| 持仓明细表 | ✅ 保留 | 和你支付宝对账 |
| 沪深300 对比 | ✅ 1 行 | 「跑输/跑赢大盘 X%」比裸数字有用 |
| 规则扫描明细 | ⚠️ 仅触发时 | 没触发则不占版面 |
| AI 全文总结 | ⚠️ 3 句以内 | 现在偏长 |
| **完整 Markdown 日报** | ❌ 建议移出邮件 | 改 GitHub Artifacts / 本地 `reports/`，邮件只留链接或「过长请查附件」 |

### 4.2 每日 **不应** 推送的

- 全市场 1.5 万只基金排行  
- 5000 分批计划全文（除非当天正是某一批执行日 → 改发 **事件邮件 C**）  
- 重复的长免责声明（页脚 1 行即可）

## 五、B. 每周配置/选基邮件（增资阶段特别有用）

**频率**：**每周 1 封**（建议周日 10:00，你有空在支付宝操作）  
**触发脚本**：`fund_recommend.py`（待接入 Actions cron 或手动）  
**主题示例**：`【选基】本周 5000 配置建议 · 宽基 50% + 主仓加仓`

### 5.1 应推送内容

| 模块 | 内容 |
|------|------|
| **本周结论** | 2 句：市场 + 对你账户的含义 |
| **推荐清单** | 最多 **3～5 只**，含代码、金额、宽基/主题标签 |
| **分配饼图文字版** | 宽基 X% / 主题 Y% / 现金 Z% |
| **分批计划** | 仅列 **下一批** 要买什么（不是三批全文） |
| **定投提醒** | 270042 继续 10 元/天，不计入本周 lump sum |
| **候选 Top 5** | 程序评分表（可选，给爱研究的人） |

### 5.2 不应推送

- 60 只候选池全表  
- 近 1 年 300%+ 基金的「排行榜炫耀」（易诱导追高）

## 六、C. 事件告警邮件（实践里最有价值、目前缺）

**频率**：**有事件才发**，与日报去重（同一天同一事件只发 1 次）

| 事件 | 优先级 | 邮件写什么 | 业界对标 |
|------|--------|------------|----------|
| **触及止损**（-15%） | 🔴 高 | 「建议评估是否减仓」+ 当前浮亏 + 持有天数 + 7 日赎回费提示 | 券商条件单 / 止盈止损提醒 |
| **触及止盈观察**（+40%） | 🟡 中 | 「可考虑部分止盈」非强制 | 目标盈提醒 |
| **分批第 N 批到期** | 🟡 中 | 「今日建议执行第 2 批：xxx 元」+ 步骤 | 定投/分批执行提醒 |
| **基金限购/暂停申购** | 🔴 高 | 代码 + 公告摘要 | 天天基金「开放提醒」 |
| **单基/主题超限**（账户 ≥1 万后） | 🟡 中 | 集中度 + 建议分散 | 组合漂移 Rebalance alert |
| **GitHub Actions 失败** | 🔴 高 | ✅ 已有 | 运维告警 |
| **净值数据异常**（缺失/跳变） | 🟡 中 | 「今日数据不可信，请勿操作」 | 数据质量 gate |

## 七、D. 月度复盘邮件

**频率**：每月 **1 封**（建议每月第二个周一 09:00）  
**主题示例**：`【月报】4 月收益 +8.2% vs 沪深300 +3.1%`

| 模块 | 内容 |
|------|------|
| 本月收益 | 金额 + 收益率 |
| vs 基准 | 沪深300、可选 270042 对标纳斯达克 |
| 执行回顾 | 定投是否坚持、分批完成了几批、AI 建议执行了几条 |
| 持仓结构 | 宽基/主题/现金占比饼图（文字） |
| 下月 1 条建议 | 如「继续日投 10 元，暂不加仓」 |
| 策略参数要不要改 | 仅提示，不自动改 yaml |

## 八、按你的投资阶段 — 该收什么、推荐什么

结合你现状：**~600 元持仓 + 日投 10 元纳斯达克 + 计划再投 5000**。

### 阶段 1：小资金积累期（账户 **< 1 万元**）← **你在这里**

| 维度 | 建议 |
|------|------|
| **邮件频率** | 日报 1 + 周日选基 0～1（增资前可每周 1） |
| **每日主推** | **持有 / 继续定投**；不因「单基 100%」逼减仓（已配置 SMALL_ACCOUNT_MODE） |
| **AI 应推荐** | ① 270042 日定投坚持 ② 有余钱再 **分批** 买 ③ 优先 **宽基打底** 再主题 |
| **暂不需要** | 复杂再平衡、换基频繁、过多只基金（≤3 只持有即可） |
| **配置重点** | `relax_caps_when_value_below: 10000` 保持 |

**本阶段邮件黄金内容（按优先级）：**

1. 今天赚/亏多少（和支付宝一致）  
2. 日定投有没有漏（270042）  
3. 若在做 5000 分批 → **只提醒「下一批」**  
4. 只有止损才红色告警  

---

### 阶段 2：增资配置期（正在投 **5000**）

| 维度 | 建议 |
|------|------|
| **邮件频率** | 日报 + **每周选基 1 封** + **分批到期事件邮件** |
| **AI 应推荐** | 012734 **加仓**（如 1000）+ **宽基 ≥50%** + 1 只互补主题 + 270042 仅定投 |
| **操作节奏** | 3 批 × 每 7 天（你 strategy 已配）；邮件只推 **当前批** |
| **核对清单** | 每批前：开放申购？限额？费率？ |

**本阶段额外推送：**

```
【分批提醒】第 2 批（T+7）
- 002900 中证500  833 元
- 012734 加仓     333 元
- 008326 通信     417 元
渠道：支付宝 → 搜索代码 → 申购
```

### 阶段 3：初具规模（**1 万～5 万**）

| 维度 | 建议 |
|------|------|
| **邮件频率** | 日报（可改为 **仅异动日** 发 full，其余发 3 行摘要）+ 月报 |
| **规则生效** | 单基 80%、主题 80% 上限开始检查 |
| **AI 应推荐** | **再平衡** > 追新基；新钱优先补 **低配宽基** |
| **持有数量** | 3～5 只足够 |

**推荐动作类型变化：**

- 少：「买新的神基」  
- 多：「A 超配减 10%，B 低配加 10%」  

---

### 阶段 4：稳定运营（**> 5 万**，远期）

| 维度 | 建议 |
|------|------|
| **日报** | 可降级为 **周报 + 事件告警**（市面高净值客户常见） |
| **重点** | 回撤控制、年度收益 vs 基准、税务/持有期（7 天赎回费） |
| **AI** | 情景分析：「若大盘跌 10%，组合预计…」 |

## 十三、总结一句话

- **每天**：只看 **持仓 + 今天要不要动 + 怎么在支付宝动**  
- **每周**（增资期）：看 **钱怎么分、下一批买啥**  
- **有事才发**：止损、分批到期、任务挂掉  
- **每月**：看 **跑得怎样、策略要不要调**  

这比「每封邮件塞完整 AI 小作文 + 全文 Markdown」更符合蚂蚁、Morningstar、Robo Advisor 的实践，也更适合你现在的 **小资金 + semi_auto** 阶段。
